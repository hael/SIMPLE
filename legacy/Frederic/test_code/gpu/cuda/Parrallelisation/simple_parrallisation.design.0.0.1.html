2. Overview

The purpose of this document is to provide the design for the SIMPLE library parallelisation scheme and the data structure put in place to amximise performance over a hybrid architecture CPU/GPU with complementary technology jsuch as OpenMP and MPI.

The NVIDIA GPU of the Tesla family with a Kelper architecture is connected via PCI-Express port. The CUDA driver 7.0 and above provides the API for the communication between the CPU and GPU.

SIMPLE optimized architecture consists of three components:
  • 3D data structure governed by the number of particles, number of rotations and the number of k.
  • Front end managed by FOTRAN 2003, as well as the main object oriented modules.
  • The GPU code is currently written in CUDA C, and will be later extended to OpenCL.
  • MPI will then be used as as data management over the all the CPU cores on the node
This document covers the design of the SIMPLE library parallelisation.

4. Functional Requirements

The requirements for SIMPLE are:
  • Re implementation of the currently hill defined data structure.
  • Implementation of 3D data structure portable to GPU with reasonable performance (Cryo-EM team: FB and HE).
  • Support of CUDA 7.0 and above with SM>= 3.5 for the GPU component. (Cryo-EM team: FB and HE)
  • Support of OpenMP for the CPU part (Cryo-EM team: FB and HE).
