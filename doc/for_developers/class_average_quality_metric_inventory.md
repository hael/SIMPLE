# Class-Average Quality Metric Reference

**Scope:** Current scalar feature bank used by `model_cavgs_rejection`.

## Purpose

This note documents the feature extraction and model inputs used by the class-average quality backend. The implementation is centralized in `src/main/cavg_quality`.

## Files

- `simple_cavg_quality_types.f90`: shared constants and derived types.
- `simple_cavg_quality_feats.f90`: feature definitions, feature extraction, hard rejects, and normalization.
- `simple_cavg_quality_stats.f90`: metrics and statistical helpers.
- `simple_cavg_quality_model.f90`: model presets, scoring, clustering, thresholding, model I/O, and promotion snippets.
- `simple_cavg_quality_analysis.f90`: analysis and feature-table output.
- `simple_cavg_quality_learn.f90`: model training from analysis files.

## Feature Bank

`CAVG_QUALITY_NFEATS` is 9. The feature order is fixed:

| Index | Feature | Source | Direction | Description |
| --- | --- | --- | --- | --- |
| 1 | `log_pop` | `cls2D` `pop` | higher is better | Log class population. |
| 2 | `neg_log_res` | `cls2D` `res` | higher is better | Negative log class resolution estimate. |
| 3 | `centered` | Otsu foreground connected components | higher is better | Negative normalized foreground centroid displacement. |
| 4 | `log_locvar_fg` | foreground mask local variance | higher is better | Log local variance inside the foreground mask. |
| 5 | `log_locvar_bg` | background mask local variance | higher is better | Log local variance outside the foreground mask. |
| 6 | `corr_frc_proxy` | `cls2D` `corr` metadata | higher is better | Stored class correlation/FRC-like proxy, when present. |
| 7 | `log_center_edge_snr` | center/edge variance statistics | higher is better | Log central variance relative to edge variance. |
| 8 | `cc_area_frac` | foreground connected-component area | higher is better | Largest foreground component area divided by expected mask area. |
| 9 | `presence` | center/border statistics | higher is better | Central signal excess relative to border background noise. |

`FEATURE_DEFS` in `simple_cavg_quality_feats.f90` defines the names, directions, and descriptions emitted in the feature inventory.

## Extraction Constants

- `FOREGROUND_SEG_LP = 30.0`
- `SIGNAL_METRIC_LP = 10.0`
- `CAVG_RES_HARD_REJECT_A = 40.0`
- `LOCVAR_WINDOW = 10`
- `MASK_HARD_OUTSIDE_PIXELS = 10`
- `LOG_EPS = 1.0e-12`

## Hard Rejects

`extract_cavg_quality_features` marks a class average as hard rejected when:

- `pop <= 0`;
- `res > CAVG_RES_HARD_REJECT_A`;
- the image contains invalid pixels;
- foreground segmentation has no valid component;
- any foreground component centroid lies outside the mask radius;
- the largest foreground component has more than `MASK_HARD_OUTSIDE_PIXELS` pixels outside the mask disc;
- foreground and background local variance are both non-positive.

Hard rejects are stored in the `hard_reject` column of analysis files and are excluded from model fitting.

## Normalization

`normalize_cavg_quality_features` normalizes each feature over the non-hard-rejected rows using robust per-dataset statistics. Normalized values are clipped by `CLIP_Z`. The model consumes normalized `z_*` features; raw values remain available in analysis outputs.

## Models

Built-in presets:

- `chunk_default_v2`
- `pool_default_v1`

The default model is `chunk_default_v2`.

The chunk model uses the 9-feature bank with these weights:

```text
log_pop             1.172054E-01
neg_log_res         1.487871E-01
centered            4.126378E-02
log_locvar_fg       1.207641E-01
log_locvar_bg       1.257626E-01
corr_frc_proxy      1.626182E-01
log_center_edge_snr 8.155017E-02
cc_area_frac        6.463601E-02
presence            1.374126E-01
```

The pool model uses `CAVG_QUALITY_POOL_V1_WEIGHTS`.

## Training Inputs

`quality_mode=learn` reads analysis files generated by `quality_mode=analyze`. Required row data:

- normalized `z_*` feature columns for all 9 features;
- `hard_reject`;
- manual class state.

Learn mode maps these fields from the current analysis-table header.

Hard-rejected rows remain visible in reports and are excluded from feature-weight estimation, candidate scoring, feature-signal diagnostics, and feature-drop diagnostics.
